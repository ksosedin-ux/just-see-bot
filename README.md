# Just See Bot — инструкция запуска

## Шаг 1. Создай бота в Telegram

1. Открой Telegram → найди **@BotFather**
2. Напиши `/newbot`
3. Придумай имя: например `Just See Growth Bot`
4. Придумай username: например `justsee_growth_bot`
5. BotFather пришлёт **токен** — сохрани его (выглядит так: `7312456789:AAF...`)

## Шаг 2. Узнай свой Telegram ID

1. Найди бота **@userinfobot**
2. Напиши ему `/start`
3. Он пришлёт твой **ID** — это число вроде `123456789`

## Шаг 3. Задеплой на Railway (бесплатно)

1. Зайди на [railway.app](https://railway.app) → войди через GitHub
2. Нажми **New Project → Deploy from GitHub repo**
3. Залей папку `just_see_bot` в GitHub (создай репозиторий)
   - Или нажми **Deploy from local** и залей папку напрямую
4. В проекте Railway → вкладка **Variables** → добавь:
   - `BOT_TOKEN` = твой токен от BotFather
   - `OWNER_ID` = твой Telegram ID (число)
5. Railway автоматически запустит бота

## Альтернатива: запуск локально (для теста)

```bash
# Установи зависимости
pip install -r requirements.txt

# Запусти с переменными
BOT_TOKEN="твой_токен" OWNER_ID="твой_id" python bot.py
```

## Как пользоваться ботом

| Команда | Что делает |
|---------|-----------|
| `/start` | Приветствие и объяснение |
| `/go` | Запустить план (сбрасывает прогресс если перезапускаешь) |
| `/tasks` | Задачи на сегодня + кнопки |
| `/progress` | Ранг, XP, стрик, прогресс по дням |
| `/stats` | История всех дней |
| `/skip` | Пропустить день (незакрытые задачи перейдут завтра) |

## Расписание

- **09:00 МСК** — утренний брифинг с задачами дня
- **13:00 МСК** — дневная проверка (сколько сделано)
- **21:00 МСК** — итог дня: XP, ранг, стрик, перенос невыполненного

## Геймификация

- **+100 XP** за каждую выполненную задачу
- **Стрик-бонус**: если закрыл все задачи дня — +50 XP × количество дней подряд
- Ранги: Новичок → Оператор → Режиссёр → Продюсер → Топ-продакшен → Just See Legend
- Невыполненные задачи автоматически переносятся в следующий день

## Если что-то не работает

- Убедись что `BOT_TOKEN` и `OWNER_ID` заданы верно
- Railway должен видеть `Procfile` в корне папки
- Логи смотри во вкладке **Deployments → Logs** в Railway
